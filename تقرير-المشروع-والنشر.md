# تقرير مشروع AL-manasa — المشاكل والحلول والنشر

## 1. ملخص المشروع

- **المشروع:** تطبيق ويب (Fusion Starter) — واجهة React مع Vite، وواجهة API بخادم Express و Prisma (SQLite).
- **الهدف:** تشغيل المشروع محلياً ونشره على منصة Render (أو مشابهة) بعد رفعه إلى GitHub.

---

## 2. المشاكل التي ظهرت والحلول المطبقة

### 2.1 خطأ استيراد `createServer` في Vite

**الرسالة:**
```text
Failed to load url /vite.config.ts. Does not export "createServer"
```

**السبب:**  
ملف `vite.config.ts` كان يستورد من `./server`، والنظام حلّ الإشارة إلى الملف الفارغ `server.ts` في الجذر بدلاً من مجلد `server/index.ts` الذي يصدر الدالة `createServer`.

**الحل:**  
تعديل السطر في `vite.config.ts` إلى:

```ts
import { createServer } from "./server/index";
```

---

### 2.2 جدول Prisma غير موجود في قاعدة البيانات

**الرسالة:**
```text
Invalid `prisma.site.findMany()` invocation:
The table `main.Site` does not exist in the current database.
```

**السبب:**  
لم تُنفَّذ ترحيلات Prisma على قاعدة SQLite، فالجداول (مثل `Site`) غير موجودة.

**الحل:**  
تشغيل ترحيلات Prisma:

```bash
npx prisma migrate deploy
```

- إنشاء ملف قاعدة البيانات (مثل `server/data/app.db` إذا كان مسارك كذلك).
- إنشاء جداول: `Site`, `Setting`, `Role`.

**ملاحظة:** إذا ظهر خطأ `EPERM` عند تشغيل `npx prisma generate` على Windows، أغلق أي عملية تستخدم Prisma (مثل خادم التطوير) ثم أعد التشغيل.

### 2.3 فشل النشر على Render — عدم اكتشاف المنفذ على 0.0.0.0

**الرسائل من Render:**
```text
Ready on http://127.0.0.1:3000
Socket.IO server running at ws://127.0.0.1:3000/api/socketio
No open ports detected on 0.0.0.0, continuing to scan...
Port scan timeout reached, no open ports detected on 0.0.0.0.
Detected open ports on localhost -- did you mean to bind one of these to 0.0.0.0?
```

**السبب:**  
الخادم كان يستمع فقط على **127.0.0.1** (localhost). منصة Render تتحقق من المنفذ على الواجهة **0.0.0.0**؛ لذلك لا تُعتبر الخدمة متاحة ويُعلَن عن فشل النشر.

**الحل المطبق في هذا المشروع:**

1. **في `server/node-build.ts`:** (لمشاريع Vite/Express)
   - الاستماع على العنوان `0.0.0.0` (أو استخدام `process.env.HOST`).
   - استخدام المنفذ من البيئة: `process.env.PORT` مع قيمة افتراضية (مثلاً 3000).

   ```ts
   const host = process.env.HOST || "0.0.0.0";
   app.listen(Number(port), host, () => { ... });
   ```

2. **ملف `server.ts` في الجذر:**  
   تم إنشاؤه ليعمل عند التشغيل بـ `tsx server.ts`، ويستمع أيضاً على **0.0.0.0** مع **process.env.PORT** حتى يتوافق مع Render. **(هذا المشروع الحالي Next.js — التعديل مطبّق في `server.ts`.)**

3. **ملف `RENDER.md`:**  
   يشرح سبب المشكلة وخطوات الربط على 0.0.0.0 واستخدام PORT، ولمشاريع Next.js إذا كان المشروع المرفوع على GitHub هو Next.js.

## 4. خطوات التشغيل المحلي بعد التعديلات

1. تثبيت الحزم:
   ```bash
   pnpm install
   ```
   أو `npm install` حسب المشروع.

2. إعداد قاعدة البيانات وترحيلات Prisma:
   ```bash
   npx prisma migrate deploy
   npx prisma generate
   ```

3. التأكد من وجود `.env` مع متغير مثل:
   ```env
   DATABASE_URL="file:./server/data/app.db"
   ```
   (يُضبط المسار حسب مكان ملف قاعدة البيانات في مشروعك؛ في مشروع Next.js غالباً `prisma/schema` وربما `file:./prisma/dev.db`.)

4. تشغيل التطوير:
   - Next.js: `npm run dev` أو `pnpm run dev`
   - Vite: `pnpm run dev`

5. للإنتاج محلياً (بعد البناء):
   ```bash
   npm run build
   npm run start
   ```
   أو تشغيل: `tsx server.ts` (مع ضبط PORT إن لزم).

## 5. النشر على Render (أو منصات مشابهة)

- **Build command (مثال):**  
  `npm run build` أو `pnpm run build` (مع `prisma generate` إن لزم، غالباً في `postinstall`).

- **Start command:**  
  - Next.js (هذا المشروع): `npm run start` → يشغّل `tsx server.ts` ويستمع على 0.0.0.0 و PORT.
  - Vite/Express: `node dist/server/node-build.mjs` أو `tsx server.ts` حسب الإعداد.

- **متغيرات البيئة المهمة:**
  - `PORT`: تُعيّنه المنصة تلقائياً؛ يجب أن يقرأه الخادم ولا يُثبت المنفذ ثابتاً.
  - `DATABASE_URL`: إذا استخدمت قاعدة بيانات خارجية (مثل PostgreSQL على Render)، ضع الرابط هنا.

- **شرط نجاح النشر:**  
  أن يستمع الخادم على **0.0.0.0** وعلى المنفذ الذي توفره المنصة في **PORT**. في هذا المشروع (Next.js) تم تطبيقه في **`server.ts`**.

## 6. إذا كان المشروع المرفوع على GitHub هو Next.js

هذا المشروع (**ciar** / nextjs_tailwind_shadcn_ts) هو Next.js. ملف الخادم المخصص **`server.ts`** في الجذر يطبّق:

- قراءة المنفذ من `process.env.PORT`.
- ربط الاستماع على **"0.0.0.0"** (عبر `process.env.HOST || '0.0.0.0'`).

تفاصيل وأمثلة في **`RENDER.md`**.

## 7. خلاصة

- تم حل مشكلة تحميل `vite.config.ts` (لمشاريع Vite) بتصحيح مسار استيراد `createServer`.
- تم حل مشكلة جدول Prisma بتشغيل `prisma migrate deploy`.
- تم حل مشكلة عدم اكتشاف المنفذ على Render بربط الخادم على **0.0.0.0** واستخدام **process.env.PORT** — في هذا المشروع عبر **`server.ts`**، وفي مشاريع Vite/Express عبر `server/node-build.ts` و `server.ts`.
- تم توثيق خطوات النشر والتشغيل في `RENDER.md` وهذا التقرير.

بعد تطبيق هذه التعديلات ورفعها إلى GitHub وإعادة النشر على Render، من المتوقع أن تمر عملية النشر بنجاح من ناحية اكتشاف المنفذ، مع مراعاة إعداد قاعدة البيانات والمتغيرات البيئية حسب خطة الاستضافة.
