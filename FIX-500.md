# إصلاح خطأ "Internal Server Error" (500)

إذا ظهرت رسالة **Internal Server Error** عند فتح `http://localhost:3000`، جرّب الخطوات التالية بالترتيب.

## 1. إعداد ملف البيئة وقاعدة البيانات

```bash
# نسخ ملف البيئة (إن لم يكن موجوداً)
copy .env.example .env

# تعديل .env وضبط DATABASE_URL، مثلاً:
# DATABASE_URL="file:./db/custom.db"
```

تأكد أن المجلد المذكور في المسار موجود (مثلاً أنشئ مجلد `db` في جذر المشروع إن استخدمت `file:./db/custom.db`).

## 2. توليد عميل Prisma وتطبيق الترحيلات

```bash
npx prisma generate
npx prisma migrate deploy
```

- إذا ظهر خطأ مثل **"table does not exist"**: نفّذ `npx prisma migrate deploy` (أو `npx prisma db push` للتطوير فقط).
- على Windows إذا ظهر **EPERM**: أغلق أي عملية تستخدم قاعدة البيانات (مثل خادم التطوير) ثم أعد التشغيل.
- إذا ظهر **ENOSPC: no space left on device**: حرّر مساحة على القرص (حذف ملفات مؤقتة، تفريش سلة المهملات، أو نقل المشروع إلى قرص آخر)، ثم أعد تشغيل `npx prisma generate`.

## 3. إعادة تشغيل الخادم

- إذا كنت تستخدم الخادم المخصص: `npm run start` أو `npx tsx server.ts`
- للتطوير: `npm run dev`

## 4. إذا استمر الخطأ

- راجع سجلات الخادم (Terminal) لمعرفة رسالة الخطأ الفعلية.
- تأكد أن المنفذ 3000 غير مستخدم من تطبيق آخر.
- راجع ملف **RENDER.md** إذا كنت تنشر على سيرفر (مثلاً Render).

بعد تطبيق هذه الخطوات، من المفترض أن تعمل الصفحة الرئيسية حتى بدون اتصال بقاعدة البيانات (ستُستخدم إعدادات ومحتوى افتراضي).
